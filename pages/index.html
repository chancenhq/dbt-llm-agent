Ragstar

Your friendly RAG agent orchestration framework.

About Ragstar

Ragstar is an advanced LLM-based AI Agent framework designed for interacting with and understanding complex projects, initially focused on dbt (data build tool). It helps teams streamline documentation, improve discovery, and gain deeper insights through natural language interaction. Leverage the power of Large Language Models to make your projects more accessible and maintainable.

Key Features

  • Natural Language Question Answering
  • Agentic Interpretation & Reasoning
  • Automated Documentation Generation
  • Semantic Search powered by Vector Embeddings
  • Integration via Manifest Files or Source Code
  • Postgres with pgvector Backend
  • Question & Feedback Tracking
  • Slack Integration
  • Extensible Architecture

Installation & Setup

Follow these steps to set up Ragstar:

  1. Check Python Version: Ensure you have Python 3.10 or higher. 
    python3 --version
  2. Clone the repository: 
    git clone https://github.com/pragunbhutani/ragstar.git
cd ragstar
  3. Install dependencies: Using Poetry. 
    # Install Poetry if needed: curl -sSL https://install.python-poetry.org | python3 -
poetry install
  4. Set up PostgreSQL: Requires PostgreSQL (11+) with the pgvector extension. 
    # Example: Local setup
createdb ragstar
psql -d ragstar -c 'CREATE EXTENSION IF NOT EXISTS vector;'
    See the pgvector docs for more options.
  5. Configure environment variables: Copy .env.example to .env and fill in your details (OPENAI_API_KEY, POSTGRES_URI, etc.). 
    cp .env.example .env
# vim .env # Or use your favorite editor
  6. Initialize the database schema: 
    poetry run ragstar init-db
  7. Initialize your project: Load metadata using one of the modes (cloud, local, source). Example using local mode: 
    # Make sure DBT_PROJECT_PATH is set in .env or provided via --project-path
poetry run ragstar init local --project-path /path/to/your/dbt/project

For full details on initialization modes and dbt Cloud setup, please refer to the README.

Using Ragstar (CLI)

Once initialized, use the CLI to interact with Ragstar:

1. Process Documentation & Embeddings:

Generate embeddings for existing docs, or interpret+embed models without docs. 

# Embed existing docs for all models
poetry run ragstar embed --select "*"

# Interpret, save, and embed models tagged 'staging'
poetry run ragstar interpret --select "tag:staging" --save --embed

2. Ask Questions:

Query your project using natural language. 

poetry run ragstar ask "Explain the fct_orders model"
poetry run ragstar ask "Which models depend on stg_users?"

3. Provide Feedback:

Help improve the agent's responses. 

# List previous questions
poetry run ragstar questions

# Mark question 1 as useful
poetry run ragstar feedback 1 --useful

# Mark question 2 as not useful and provide correction
poetry run ragstar feedback 2 --not-useful --text "The upstream source is incorrect."

4. Other Commands:

# List models
poetry run ragstar list

# Get details for a specific model
poetry run ragstar model-details my_model_name

This covers the main CLI commands. Refer to the README for more advanced options and programmatic usage.

Slackbot Integration

Interact with your Ragstar agents directly from Slack:

  1. Configure Slack API tokens (SLACK_BOT_TOKEN, SLACK_APP_TOKEN) in your .env file.
  2. Run the Slackbot application:
poetry run python -m ragstar.integrations.slackbot

Mention the bot in your Slack workspace to start interacting.
